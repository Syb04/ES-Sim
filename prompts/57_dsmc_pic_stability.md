# 57: DSMC-PIC 連成の数値発散対策 — 設計メモ

ユーザーの CEL ホール形状 (0.8 mm 孔から 100 Pa 供給 → 5 Pa 排気、rz_x0、
孔周り 0.05 mm 局所メッシュ = 118k 要素) で PIC が発散して NaN 入り JSON を
送っていた件の調査と対策の記録。

## 実測から判明した3つの問題と対策

1. **NaN 入りフレームの送出** → run_batch で毎ステップ φ・KE の有限性を検査し、
   非有限なら対処方針つきの ValueError で停止 (WS の error として届く)

2. **DSMC のマクロ重みが最大リザーバ密度基準** → 500,000 粒子指定でも初期充填
   (5 Pa) では実質 25,000 個 (100 Pa 基準の 1/20)。118k セルに対して
   セルあたり 0.2 個で密度スパイクが物理値の2倍立っていた
   → **初期充填密度基準に変更** (500k 指定が実際に 500k になる)。
   実測: 150k×avg2000 でスパイク +51% → 500k×avg3000 外挿で +23%

3. **null-collision の ν_max がテーブル全域 (〜8 keV) で過大評価** →
   励起テーブルが 4000 eV まであるため ν_max = 8.7e10 (実在する 〜200 eV 域は
   1.2〜2e10)。衝突候補率がほぼ1になり dt 選定の指標も壊れていた
   → **適応 ν_max**: 毎呼び出し、現在の粒子最大エネルギーまでの prefix max で
   ν_max を評価 (ν_max ≥ 全粒子の ν(E) は保たれるので null-collision は厳密なまま。
   com 系イオンはガス熱速度 6σ + 流速のマージン)。起動時警告も代表 200 eV 域で評価

## このケースの推奨条件 (実測ベース)

- DSMC: n_particles 500,000 (W修正後)、n_steps 10,000 (定常到達 ≈ 4,850)、
  avg_steps 3,000 → スパイク +20%程度、<30サンプルセルほぼ0
- PIC: **dt = 3e-11 s を明示指定** (自動 1.77e-10 では候補率 ~0.9)。
  候補率 0.3〜0.45、RF 1.2 周期の安定完走を確認。RF 1周期 ≈ 2,460 ステップ
