# サブエージェント指示: RF1周期の位相分解データ出力(バックエンド)

対象リポジトリ: `/home/claude/ES-Sim/backend`(pic.py の最新実装が前提)

PIC 完了時に、RF の1周期分を位相分解した時系列フィールド(アニメーション用)を返す。

## スキーマ契約(フロントと共通)

- `PicSettings` に `phase_bins: int = 40` を追加(0 で無効)
- WS `done` に `cycle` を追加(RF が未設定、または phase_bins=0 の場合は省略):

```jsonc
"cycle": {
  "bins": 40,
  "period_s": 7.37e-8,               // RF周期
  "phi":  [[...], ...],              // bins × 節点  位相分解平均の電位 [V]
  "n_e":  [[...], ...],              // bins × 節点  同 電子密度 [m^-3]
  "n_i":  [[...], ...],              // bins × 節点  同 イオン密度 [m^-3]
  "particles": {                     // 最後の1周期の生スナップショット (位相ビンごと)
    "electron": [[[x,y],...], ...],  // bins × ≤1000点 (間引き)
    "ion":      [[[x,y],...], ...]
  }
}
```

## 実装

- RF 周波数は boundaries / conductor 領域の voltage_rf から最初に見つかったものを使用
  (複数あれば最初の1つ。無ければ cycle 機能は無効)
- **位相分解平均**: 既存の平均区間 (avg_steps) の各ステップを位相ビン
  `bin = floor((t mod T)/T × bins)` に割り当て、φ・n_e・n_i をビンごとに積算・平均
  (ビンごとのステップ数で割る。ステップ数0のビンは前後から補間せずゼロで良いが、
  avg_steps が1周期以上あれば通常発生しない。avg_steps < 1周期分なら警告を started に追加)
- **粒子スナップショット**: 実行の最後の1周期の間、各位相ビンに最初に該当したステップで
  種ごとに最大1000点へ間引いた位置を保存
- ペイロード削減: 数値は float32 精度相当に丸めて JSON 化(例: `np.float32` 経由)
- 平均区間外のステップ単価を悪化させないこと

## テスト(`tests/test_cycle.py` 新規)

1. RF付き小ケースで cycle が返り、bins×節点の寸法・period_s が正しいこと。
   φ の位相依存: RF電極近傍節点の位相分解 φ がビン間で振動し、
   その振幅が RF 振幅のオーダーであること(0.2×〜1.2×)
2. 位相平均の整合: cycle.phi の全ビン平均が fields.phi と数%以内で一致
3. RFなし・phase_bins=0 では cycle が省略されること
4. 既存59テストを壊さない
