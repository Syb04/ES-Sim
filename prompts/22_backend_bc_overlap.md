## A. 境界条件タイプの追加(スキーマ契約、フロントと共通)

```jsonc
BoundaryCondition:
  edges: [int]
  type: "dirichlet" | "symmetry" | "periodic"
  voltage / voltage_rf / see_gamma   // dirichlet のみ有効
```

- **symmetry(対称)**: 場は自然境界(Neumann、現状の未指定と同じ)。
  粒子(/trace のフェーズ2粒子・PIC の全種・SEE電子)は**鏡面反射**。
  既存の `pic.reflect_edges` は残し、内部では symmetry 指定エッジとの和集合として扱う
- **periodic(周期)**: 1つのBCエントリに **edges をちょうど2本**指定(平行・同長の対辺。
  validator で検査、矩形domainの対辺を想定)。
  - **場**: 対辺の節点DOFを同一視して解く。gmsh の `mesh.setPeriodic`(次元1、平行移動の
    アフィン変換)で対辺のメッシュを一致させ、スレーブ節点→マスター節点の対応を取得。
    組み立て時に triangles の節点番号を正準化(スレーブ→マスター置換)してから既存の
    組み立て・求解を使い、解ベクトルはスレーブ節点へコピーして返す(表示互換)
  - **粒子**(trace・PIC): 周期エッジを越えた粒子は位置を周期ベクトル分平行移動して
    反対側へラップ(速度不変)。所属要素はラップした少数粒子のみ `_locate_initial`(総当たり)で
    再特定すれば良い。壁カウンタ非計上
  - 電荷堆積・密度積算は正準化された節点番号を使えば自動的に整合する

## B. 外枠に重なる領域(メッシュ生成の OCC 化)

現状の meshing.py は「領域は domain の内部に完全に含まれる」前提(geo カーネルで穴を開ける)
のため、外枠の線に重なる/はみ出す矩形が扱えない。**gmsh OCC カーネル + boolean fragment**
ベースに書き換える:

1. domain・各領域のポリゴン/円を occ で surface 化し、`occ.fragment` で全て分割
2. 各フラグメント面の内部点(`getCenterOfMass` 等)がどの領域ポリゴンに含まれるかで
   面→領域を同定(domain 外のフラグメントは破棄、conductor 内は穴として面を除外)
3. conductor の Dirichlet 節点: conductor 領域に隣接する境界曲線上の節点
   (フラグメント後の曲線をジオメトリで判定)
4. 外周エッジのBC割り当て: フラグメント後の外周曲線の中点が元の domain エッジ i 上に
   あるかで対応付け(電極が外枠に重なった区間は電極の Dirichlet を優先)
5. 局所メッシュサイズ(local_sizes)は従来相当の挙動を維持
6. 領域がはみ出した部分は**黙って domain にクリップ**される仕様とする

既存機能の回帰確認: 平行平板・同軸円筒(256角形domain)・circle shape・Turnerストリップの
各テストが**全て従来と同等の結果**になること(メッシュ自体は変わり得るので、
解析解比較テストの許容誤差内でパスすれば良い)。

## C. テスト追加(`tests/test_bc_overlap.py` 新規)

1. **境界重なり**: 平行平板 domain の左辺に重なる矩形電極(半分はみ出し)でメッシュ生成が
   成功し、電極輪郭に Dirichlet が付き、/solve が妥当な V 範囲を返す
2. **対称性**: 全平行平板と、対称辺で半分にした問題の解が対称軸上で一致(相対誤差 <1e-8)
3. **周期(場)**: 上下周期の平行平板(左右Dirichlet)で、上下対辺の節点電位が一致し、
   解が x のみの関数(解析解と一致)
4. **周期(粒子)**: /trace で上下周期の場を横切る斜め発射粒子が y をラップして進み、
   吸収されないこと(x方向の物理は解析解通り)
5. **対称(粒子)**: symmetry 辺で /trace 粒子が反射すること
