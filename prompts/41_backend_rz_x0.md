# サブエージェント指示: 左辺 (x=0) を軸とする軸対称モード(バックエンド)

対象リポジトリ: `/home/claude/ES-Sim/backend`(prompts/39 の rz 実装が前提)

既存の rz モード(下辺 y=0 が対称軸、x=z・y=r)に加え、**左辺 x=0 を対称軸とする**
軸対称モードを追加する。

## スキーマ契約(フロントと共通)

- `Project.coord: Literal["xy", "rz", "rz_x0"]`
  - `"rz"`: 従来通り(y=r、対称軸は y=0)
  - `"rz_x0"`: **x=r(径方向)、y=z(軸方向)**、対称軸は x=0
- validator(rz_x0): domain 全頂点 x ≥ 0、x=0 上の辺への Dirichlet 禁止(既存 rz の検査を
  径方向インデックスで一般化)

## 実装

既存の rz 分岐を「径方向座標インデックス ridx(rz: 1、rz_x0: 0)」で一般化するのが方針。
コピペ分岐を増やさないこと。

- fem.py: r 値の取得を nodes[:, ridx] に一般化(剛性の r̄、電荷右辺、エネルギー 2πr)
- particles.py(trace): 速度成分の対応を一般化 —
  rz: (vx,vy,vz)→(v_z軸, v_r, v_θ)、rz_x0: (vx,vy,vz)→(v_r, v_z軸, v_θ)。
  遠心力項は径方向成分に、角運動量 L=r·vθ、軸交差の鏡映は径座標・径速度に適用
- pic.py: rz_x0 も未対応エラーの対象(メッセージは「軸対称」で共通に)
- schema/server: coord 透過

## テスト(`tests/test_rz_x0.py` 新規)

1. **円筒コンデンサ(縦向き)**: domain [a,b]×[0,L](左辺 x=a に V1、右辺 x=b に 0V)で
   V(r)=V1·ln(b/r)/ln(b/a) と一致(<1e-2)、容量 C=2πεL/ln(b/a) と数%以内
2. **rz との等価性**: 同一の物理(90°回転した設定)で rz と rz_x0 の中心付近の解・容量が
   数値的にほぼ一致すること
3. **粒子**: vθ≠0 の自由粒子で L 保存・軌道解析解一致(rz テストの回転版)。軸(x=0)交差の鏡映
4. **バリデーション**: x<0 domain・x=0 辺への Dirichlet・pic 実行のエラー
5. 既存87テストを壊さない(xy・rz 経路は数値不変)
