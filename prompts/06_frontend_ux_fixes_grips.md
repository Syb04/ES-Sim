# サブエージェント指示: 選択感度改善・ルーラー・カラーバー修正・頂点グリップ編集

対象リポジトリ: `/home/claude/ES-Sim/frontend`(npm install 済み)

ES-Sim は 2D静電場FEMアプリ(Tauri 2 + React 18 + TS strict)。主対象は
`src/canvas/CadCanvas.tsx` と `src/App.tsx`。必ず現状のコードを読んでから着手すること。
ユーザーからの動作確認フィードバックに基づく修正4件。

## 1. 選択クリックの感度改善(不具合修正)

「選択ツールでオブジェクトをクリックしても反応しない時がある」との報告。以下を実装:

- **クリック判定の統一**: mousedown→mouseup の画面上移動量が 5px 未満なら「クリック」とみなし、
  ドラッグ開始判定に食われていても必ず選択処理を行う
- **ヒットテストに許容誤差**: 点内包判定に加え、ポリゴンの**エッジから画面上 6px 以内**
  (ワールド換算 6/scale)も命中とする(輪郭線の上をクリックしても選択できるように)
- 複数領域が命中した場合は**面積の小さい方**を優先(入れ子でも内側を選べるように)
- 選択中領域の内側を押してわずかに動いた場合(<5px)は移動ではなくクリック扱いにする

## 2. キャンバスのルーラー(目盛り)

- キャンバスの**上辺と左辺**に常時表示のルーラー(幅 ~24px)を追加。同じ canvas に直描きで良い
- 目盛り間隔は既存のグリッド自動ステップ(`gridStep`)と揃え、主目盛りに mm 単位の数値ラベル、
  その 1/10 に短い副目盛り(ズームで自動追従)
- カーソル位置を示すマーカー線をルーラー上に表示
- 左上コーナーは小さな塗りつぶし矩形で埋める。配色は既存ダークテーマの CSS 変数に合わせる
- 座標表示・パン/ズーム・作図の各操作がルーラー領域の分だけずれないよう注意
  (ルーラーは描画オーバーレイとし、マウス座標系は従来のままで良い)

## 3. カラーバーの数値ラベル見切れ修正(不具合修正)

「|E| 表示のときカラーバーの数値ラベルが見切れる」との報告。

- `ctx.measureText` で幅を測り、キャンバス右端からはみ出さないよう右揃え・マージン確保で配置し直す
- 大きい値(例 1.2e+4)は指数表記 `toExponential(1)` 等で桁を抑える
- V 表示・|E| 表示の両モードで確認すること

## 4. 頂点グリップ編集

選択ツールで領域を選択中に:

- **頂点ハンドル**: 各頂点に小さな矩形ハンドル(画面上 ~7px)を表示。ドラッグで頂点を移動
  (グリッドスナップ適用、Esc でキャンセル)。mouseup で確定し App に通知 → 履歴1エントリ
- **中点ハンドル**: 各エッジ中点に小さな円ハンドルを表示。ドラッグすると新しい頂点として挿入され
  そのまま移動できる(確定で履歴1エントリ)
- **頂点の削除**: 頂点ハンドルをダブルクリックで削除(頂点数が3未満になる場合は無視)
- ヒットテストの優先順位: 頂点ハンドル > 中点ハンドル > 領域移動 > 選択
- App 側には汎用の `onEditRegionPolygon(id, polygon: Point[])` を追加して使う(履歴を積み、結果破棄)
