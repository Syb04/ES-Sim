# サブエージェント指示: 誘電体の二次電子放出(バックエンド)

対象リポジトリ: `/home/claude/ES-Sim/backend`(pic.py の最新実装が前提)

誘電体(dielectric)領域にも二次電子放出係数 γ を設定できるようにする。
現在 γ は conductor 領域と Dirichlet 境界辺のみ(境界メッシュエッジ経由の SEE)。
誘電体は「固体要素への進入による吸収」経路(prompts/24)なので、そこに SEE を追加する。

## 仕様

- `Region.see_gamma` は既存フィールドをそのまま使う(dielectric でも有効に。スキーマ変更が
  必要なら validator の許可条件のみ調整)
- **PIC**: イオンが誘電体要素へ進入して吸収された際、その領域の γ>0 なら確率 γ で
  SEE 電子を1個生成する:
  - 位置: 吸収イオンの**プッシュ前の位置**(直前までいたプラズマ側の要素内。所属要素も
    プッシュ前のものを使えるので locate 不要)
  - 速度: 入射イオン速度の単位ベクトルの**逆向き**に see_energy_ev(法線が無い固体吸収
    経路での近似であることをコメントに明記)。面内2成分+vz は入射の逆
  - 重み: 吸収イオンと同じ
- **表面電荷の収支整合**(重要): SEE 電子が表面から放出される分、表面は正に帯電する。
  SEE 発生時に **+e·w を Q_surf へ加算**(位置はイオン吸収と同じ P1 射影)。
  ※電極 γ 由来の既存 SEE は表面電荷管理外なので変更しない
- 診断 see_events は誘電体 SEE も加算
- 電子の誘電体吸収では SEE を発生させない(電極と同様、イオン誘起のみ)

## テスト(`tests/test_dielectric_see.py` 新規)

1. γ=1 の誘電体スラブへイオンビームを打ち込み: 吸収イオン数 = SEE電子生成数、
   SEE電子がプラズマ側(スラブ外)の要素に生成され速度が入射と逆向き成分を持つこと
2. **電荷収支**: γ=1 で吸収完了後の Q_surf.sum() = Σw·(+q_ion) + Σw·(+e)
   (イオン電荷+SEE放出分。rel 1e-12)。γ=0 では従来通りイオン電荷のみ
3. γ=0(未設定)の誘電体では SEE が発生しないこと(既存挙動不変)
4. 既存77テストを壊さない
