対象リポジトリ: `/home/claude/ES-Sim/backend`(最新実装が前提)

静電場ソルバー(フェーズ1)と粒子軌道追跡(フェーズ2)に軸対称モードを追加する。
**PIC は今回対象外**(rz + pic 実行は明確な日本語エラー)。

## 座標系の約束(フロントと共通)

- `Project.coord: "xy" | "rz" = "xy"` を追加(トップレベル)
- rz モード: **x = z(軸方向)、y = r(径方向)**。下辺 y=0 が対称軸。
  validator: rz では domain の全頂点が y ≥ 0 であること。y=0 の辺(エッジ0等)への
  Dirichlet 指定は禁止(対称軸は自然境界)— 明確なエラー

## 実装

### fem.py(軸対称の弱形式)

- ∇·(ε∇V) = −ρ の軸対称弱形式: ∫ ε ∇V·∇W **r** dr dz = ∫ ρ W **r** dr dz
- P1 剛性: 平面の Ke に要素重心半径 r̄ = (r_i+r_j+r_k)/3 を乗じる(標準近似で可。
  ただし r̄ を使う旨コメント)
- 電荷右辺: f_i = ρ ∫ N_i r dA を**厳密に**線形 r で積分(∫N_i r dA = (A/12)(2r_i + r_j + r_k))
- E = −∇V は従来通り(要素定数)。エネルギー: W = ½∫ε|E|²·2πr dA **[J]**
  (xy モードは従来通り [J/m]。Solution にどちらの単位かは呼び出し側が coord で判断)
- 軸上 (r=0) の要素も r̄>0 なので特異なし。周期・対称境界は従来通り機能すること
  (rz では通常使わないが壊さない)

### particles.py(軸対称の軌道追跡)

- 状態: 位置 (z, r) = 既存の (x, y)、速度 3成分を (vz, vr, vθ) と解釈
- プッシュ: E=(Ez, Er) による加速 + **角運動量保存**: L = r·vθ を初期に確定し、
  ステップ後に vθ = L/r で更新、遠心力項は運動方程式 dvr/dt = qEr/m + vθ²/r を
  リープフロッグに組み込む(vθ²/r は現在位置で評価する半陰的で可。実装方式をコメントに明記)
- **軸交差**: r < 0 になったら r → −r、vr → −vr、vθ → −vθ(位置・速度の鏡映)。
  所属要素は walk で追従(反射後に再 walk)
- L=0(vθ=0)の粒子は従来の面内運動と同等になること
- エミッタの Maxwell 分布の第3成分は vθ として扱う

### server / schema

- /solve /trace は coord を透過的に処理。/profile も動作(座標はそのまま)
- pic + rz は 422(「PICは軸対称モード未対応」)

## テスト(`tests/test_axisymmetric.py` 新規)

1. **円筒コンデンサ**(domain 矩形 [0,L]×[a,b]、下辺 r=a に V1・上辺 r=b に 0V、左右対称境界
   または Neumann): V(r) = V1·ln(b/r)/ln(b/a) と一致(節点相対誤差 <1e-2、メッシュ細分で改善)。
   平面モードでは線形になることも対比確認(= r重みが効いている証拠)
2. **エネルギー/容量**: 同ケースで W = ½·C·V1²、C = 2πεL/ln(b/a) と数%以内
3. **軸を含む解**: y=0 を含む domain(軸対称の平行平板 = 円板コンデンサの中心部)で
   解が有限・軸上で ∂V/∂r ≈ 0
4. **粒子**: 一様 Ez 場での加速が xy モードと同一(vθ=0)。vθ≠0 の粒子で
   L=r·vθ が保存(rel 1e-6)し、全エネルギーが保存(rel 1e-3 程度)。軸交差の鏡映動作
5. **バリデーション**: rz で y<0 domain・軸への Dirichlet・pic 実行がそれぞれ明確なエラー
6. 既存80テストを壊さない(xy 経路は数値不変)
