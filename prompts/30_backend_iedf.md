# サブエージェント指示: IEDF/IADF コレクタ(バックエンド)

対象リポジトリ: `/home/claude/ES-Sim/backend`(pic.py の最新実装が前提)

指定した境界範囲(コレクタ線分)に衝突したイオンのエネルギー・入射角・重みを PIC 実行中に
記録し、done で返す。ウエハ面の IEDF/IADF 取得が目的。

## スキーマ契約(フロントと共通)

- `PicSettings` に追加:

```jsonc
"collector": {              // null なら無効
  "p1": [x, y], "p2": [x, y],   // コレクタ線分 [m]
  "tol": null                   // 判定距離 [m]。null なら mesh.size と同値
}
```

- WS `done` に追加(collector 有効時のみ):

```jsonc
"collector": {
  "count": 12345,               // 記録したマクロイオン数 (総数)
  "total_weight": 1.2e13,       // 総実イオン数 [1/m]
  "energies_ev": [...],         // サンプル (上限 50000。超過分は数のみ計上)
  "angles_deg": [...],          // 同数。符号付き入射角
  "weights": [...],             // 同数
  "truncated": false
}
```

## 実装

- **対象**: 平均区間 (avg_steps) 中に吸収されたイオン(外周・電極輪郭・誘電体表面のすべて)。
  吸収位置(既存の補間位置)がコレクタ線分から距離 tol 以内、かつ線分区間内
  (端点から線分方向への射影が [0, |p2-p1|])のもの
- **エネルギー**: 衝突時速度(3成分)から ½m|v|²/e [eV]
- **入射角**: コレクタ線分の法線に対する速度の角度 [deg]。符号は接線 (p1→p2) 方向成分の
  符号とし、範囲 (-90, 90)。法線の向きは「入射イオンの速度と逆向き成分を持つ側」
  (=表面に向かってくる側) を正面とする
- 記録は numpy 配列のチャンク追記等で高速に。上限 50000 サンプル
  (到達後は count/total_weight のみ加算し truncated=true)
- SEE で生成された電子や電子の衝突は対象外(イオンのみ)
- run_batch でも同データを `self.collector_result` に格納

## テスト(`tests/test_iedf.py` 新規)

1. **決定的ケース**: 平行平板 DC (陰極 -100V)、無衝突、上下対称境界、低温イオンを
   陰極前から静かに放して加速 → 陰極面コレクタで全イオンが記録され、
   エネルギーが電位差から期待される値 (~発射位置の電位 [eV]) と数%以内、
   入射角がほぼ 0°(|角| < 5°)であること
2. **範囲判定**: コレクタ線分を陰極面の一部 (中央半分) にした場合、記録数が全面の場合より
   少なく、記録された吸収位置に対応する y 範囲が線分区間内であること
   (吸収位置の再構成が難しければ、線分を面の外に置いたとき記録0になることの確認でも可)
3. **重み整合**: total_weight = Σ weights (truncated=false 時)
4. 既存62テストを壊さない(collector=null 経路は数値不変)
