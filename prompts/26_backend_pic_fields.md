# サブエージェント指示: PIC結果フィールドの時間平均出力(バックエンド)

対象リポジトリ: `/home/claude/ES-Sim/backend`(pic.py の最新実装が前提)

PIC 完了時に、時間平均した2Dフィールド一式をフロントへ返し、プロット可能にする。

## スキーマ契約(フロントと共通)

- `PicSettings` に `avg_steps: int | None = None` を追加
  (時間平均する最終ステップ数。None なら全ステップの最後の25%)
- WS `done` メッセージに `fields` を追加:

```jsonc
"fields": {
  "phi":      [...],  // 節点、時間平均電位 [V]
  "e_abs":    [...],  // 要素、時間平均 |E| [V/m] (平均は E ベクトルを平均してから絶対値)
  "n_e":      [...],  // 節点、電子密度 [m^-3]
  "n_i":      [...],  // 節点、イオン密度 [m^-3]
  "te_ev":    [...],  // 節点、電子温度 [eV] = (2/3)×平均運動エネルギー (3速度成分)。粒子なし節点は 0
  "ion_rate": [...],  // 節点、電離レート [m^-3 s^-1] (電離イベント位置をP1重みで散布し、時間・節点集中面積で規格化)
  "avg_steps": 12800  // 実際に平均したステップ数
}
```

## 実装

- 既存の密度アキュムレータを拡張し、平均区間で毎ステップ以下を積算:
  - φ(節点)、E ベクトル(要素)
  - 種別ごとの重み密度(既存)+ 電子の Σ w·L_i·(½m|v|²)(節点)→ 平均後に
    Te[eV] = (2/3)·(エネルギー和/重み和)/e。重み和 0 の節点は 0
  - 電離イベント発生位置での P1 散布(重み w)→ ion_rate = 積算 / (平均時間 × 節点集中面積)
- WS 実行時: 開始時に avg_steps を決めてアキュムレータを有効化し、done で fields を送出
- run_batch でも同じ fields を返す(検証スクリプトの将来利用のため)
- 追加コストは平均区間のみで、平均区間外のステップ単価を悪化させないこと

## テスト(`tests/test_pic_fields.py` 新規)

1. 平行平板+初期プラズマ(衝突なし)で: phi 平均が有限で境界値と整合、n_e/n_i が
   密度アキュムレータと一致、Te が初期温度のオーダー(0.5×〜2×)にあること
2. MCC電離ありの小ケースで: ion_rate の総和×体積×時間 ≒ 電離イベント数×重み(rel 5%以内)
3. avg_steps 指定/None(25%既定)の両方で動作
4. 既存55テストを壊さない
