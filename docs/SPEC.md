# ES-Sim 仕様書

2Dスケッチ CAD + 静電場シミュレーションアプリケーション

- 版: 0.1(初版・対話による仕様決定の結果)
- 日付: 2026-07-21
- 用途: 個人の研究・検討ツール(配布は当面考慮しない)

---

## 1. 概要

2次元平面上でジオメトリをスケッチし、そのまま静電場解析にかけられるデスクトップアプリケーション。
解析は3段階で拡張する。

1. **フェーズ1 — 静電場ソルバー**: 電極・誘電体・空間電荷を配置してポアソン方程式を FEM(三角形1次要素)で解き、電位 V と電場 E を可視化する
2. **フェーズ2 — 荷電粒子軌道追跡**: 解いた静電場(固定場)の中で電子・イオンの軌道を積分する
3. **フェーズ3 — PIC**: 粒子の電荷を場にフィードバックする自己無撞着な時間発展(FEM-PIC)

### 参考実装との対応

| 参考 | 取り入れる要素 |
|---|---|
| [KulmanLab](https://github.com/volodymyr4509/kulmanlab) | ブラウザ技術によるCADキャンバスUX(スナップ、グリッド、コマンド操作) |
| [FEMLab](https://github.com/enizimus/FEMLab) | 三角形要素FEM、V/E/D可視化、境界条件の扱い、ポスト処理の項目立て |
| [PyPIC3D](https://github.com/uwplasma/PyPIC3D) | Boris/リープフロッグ粒子プッシュ、診断出力、設定駆動の実行方式 |

## 2. 決定事項サマリ(対話で確定)

| 論点 | 決定 |
|---|---|
| 実行形態 | デスクトップアプリ(Windows) |
| フロント | Tauri 2 + React + TypeScript(Canvas 2D 描画) |
| バックエンド | Python(FastAPI ローカルサーバー、サイドカープロセス) |
| 解析範囲 | 静電場FEM → 粒子軌道 → PIC の3段階 |
| 数値解法 | FEM / 非構造三角形メッシュ(1次要素から) |
| 座標系 | 平面2D(x-y、奥行き一様)のみ。軸対称は将来拡張 |
| CAD規模 | シミュ用簡易スケッチ(線・ポリライン・矩形・円・円弧+スナップ) |
| 保存形式 | 独自JSONプロジェクトファイル |
| 計算基盤 | **numpy/scipy 基盤 + CuPy によるGPUオプション**(下記注記) |

### 計算基盤に関する注記(JAXからの変更)

当初 JAX(GPU)を候補としたが、JAX の GPU 実行は 2026年7月現在も Linux / WSL2 限定であり、
Windows ネイティブの CUDA 対応は未サポート(jax-ml/jax issue #32741、公式インストールガイド)。
「WSL2 は使いたくない」という制約と両立しないため、次の方針とする。

- **基盤**: numpy + scipy.sparse(全機能がCPUで完結して動作すること)
- **GPUオプション**: 粒子プッシュ・電荷堆積などのホットループを CuPy に差し替え可能にする
  (CuPy は NVIDIA CUDA で Windows ネイティブ動作。AMD GPU は Windows では対象外)
- **抽象化**: 配列モジュールを `es_sim/backend.py` の `get_xp()` 経由で取得し、
  numpy / cupy を実行時に切り替える。将来 Linux 環境に移す場合は JAX 実装の追加も可能

FEM の疎行列直接法ソルブ(スパースLU)は CPU の方が実装・安定性ともに有利であり、
2Dの静電場(〜数十万要素)なら scipy で十分高速。GPUが効くのは粒子系(フェーズ2・3)。

## 4. CAD仕様(簡易スケッチ)

### 4.1 描画プリミティブ

- 線分、ポリライン(開/閉)、矩形、円、円弧
- 円は「中心+半径」のパラメトリック形状(`shape: {kind:"circle"}`)として保存し、
  **メッシュ生成時に** `n = clamp(ceil(2πr/h), 24, 720)`(h=その領域の特性長)で多角形化する。
  頂点間隔がメッシュサイズの下限になる問題を避けるため、固定分割の polygon 化はしない。円弧は将来対応

### 4.2 操作系

- グリッド表示・グリッドスナップ、端点/中点スナップ
- パン(中ボタン/Space+ドラッグ)、ホイールズーム
- 選択(クリック/矩形)、移動、削除
- 数値入力(座標・長さ)は座標表示+入力欄で対応(コマンドライン入力は将来)
- Undo/Redo(操作履歴スタック、プロジェクト状態のスナップショット方式で開始)

### 4.3 モデル構造

CADの目的は「閉領域と境界エッジに物理属性を付けること」。エンティティは次の3種。

- **domain**: 解析領域の外周(閉ポリゴン1つ。必須)
- **region**: 領域内の閉図形(電極・誘電体・空間電荷のいずれかの材料を割り当て)
- **boundary edge**: domain 外周や region 輪郭のエッジ列(境界条件を割り当て)

材料・境界条件:

| 種別 | パラメータ | FEM上の扱い |
|---|---|---|
| 電極 (conductor) | 電位 V [V] | 領域輪郭の節点に Dirichlet(領域内部はメッシュから除外) |
| 誘電体 (dielectric) | 比誘電率 εr | 要素ごとの係数 ε = ε0·εr |
| 空間電荷 (charge) | 電荷密度 ρ [C/m³] | 右辺ベクトルへ寄与 |
| 真空/空気 | εr = 1 | デフォルト材料 |
| 外周境界 | Dirichlet(V指定) / Neumann(自然境界=対称面) | エッジ単位で指定。未指定は Neumann |

### 4.4 単位系

SI(長さm、電位V、電荷C)。UI上は mm 表示の切り替えを持つ(内部は常にm)。

## 5. メッシュ生成

- gmsh(公式 Python API)を使用。ポリゴン群 → Plane Surface → 三角形メッシュ
- 電極領域は穴として抜き、輪郭に物理グループ(Dirichletタグ)を付与
- 誘電体・電荷領域は埋め込みサーフェスとして領域タグを保持
- メッシュサイズ: 全体特性長 + 領域/エッジ単位のローカルサイズ指定
- 品質・要素数をメッシュプレビューで確認できること(/mesh を solve と分離する理由)

## 6. FEMソルバー(フェーズ1)

- 定式化: ∇·(ε∇V) = −ρ の弱形式、P1(線形三角形)要素
- 組み立て: 全要素一括のベクトル化組み立て(要素ループをnumpyで排除)→ scipy.sparse.coo → csr
- Dirichlet: 行削除方式(縮約)または大数法ではなく、対称性を保つ縮約方式を採用
- 求解: scipy.sparse.linalg.splu(直接法)。行列を保持し、右辺のみ変わる再解析(電位変更、PICの毎ステップ)は再分解なしで解く
- 導出量: E = −∇V(要素定数)→ 節点平均で滑らかな表示用フィールドも生成
- 検証ケース(自動テスト):
  - 平行平板: V線形分布・E一様(解析解と比較、相対誤差 < 1e-10)
  - 同軸円筒(2D断面): V(r) の対数分布(メッシュ収束で確認)
  - 誘電体境界: D法線連続の確認

## 7. 可視化・ポスト処理(フェーズ1)

- 電位 V: カラーマップ(コンター)、等電位線
- 電場: |E| カラーマップ、ベクトル矢印、電気力線(始点指定のストリームライン)
- メッシュ表示のオン/オフ
- ラインプロファイル: 2点間の V / |E| をサンプリングしてグラフ表示・CSV出力
- 静電容量・蓄積エネルギー(W = ½∫ε|E|²dΩ)の算出は容易なので早期に入れる
- 画像(PNG)エクスポート

## 8. 粒子軌道追跡(フェーズ2)

- 前提: フェーズ1の場を固定場として使用
- 粒子種: 電荷 q・質量 m を指定(電子・陽子プリセット)。非相対論、2D面内運動
- 積分器: リープフロッグ(静電場のみなのでBorisの磁場回転項は省略。B追加時にBoris化)
- 場の補間: 粒子位置を含む三角形を特定(walk探索+前回要素キャッシュ)→ 重心座標で E を補間(P1なら要素内一定)
- 粒子ソース: 点/線分エミッタ。エネルギー分布は単一エネルギー(mono)+角度広がり、
  または **Maxwell分布**(ドリフト+2D熱速度、温度 kT [eV]・乱数シード指定で再現可能)
- 境界イベント: 電極・外周に到達したら吸収(衝突位置・エネルギー・**入射角**を記録)
- 出力: 軌道のオーバーレイ表示、着地点分布、飛行時間
- ホットループは numpy ベクトル化(全粒子一括)。CuPy 切り替えの最初の適用対象

## 9. PIC(フェーズ3)

- 方式: FEM-PIC(非構造三角形メッシュ上のPIC)。構造格子PICより電荷堆積・粒子位置特定が難しい点は認識済み
- 代表ターゲット: **低温プラズマ(CCP)**。電極・境界の電圧に RF 重畳
  `V(t) = V_dc + A·sin(2πft + φ)`(`voltage_rf` フィールド)を指定可能。
  剛性行列は不変なので前分解LUを保持し、Dirichlet値のみ毎ステップ更新する
- 粒子種: 電子+イオン(質量 amu 指定、検証用にイオン固定オプション)の2種を常時管理
- **MCC衝突**(実装済み): 背景中性ガス(一様圧力[Pa]・温度[K]指定)との null-collision 法。
  電子: 弾性(2m/M損失)・励起(閾値損失)・電離(新e+新イオン生成、余剰エネルギー乱数分配)。
  イオン: 等方弾性・後方散乱(電荷交換)。断面積は **LXCat形式ファイルをインポート**
  (標準ブロック形式と、Phelpsイオンデータ等のタイプ行なし SPECIES/PROCESS 形式の両対応)し、
  パース済みテーブルをプロジェクトJSONへ埋め込む(`POST /lxcat/parse`)
- **二次電子放出(SEE)**: 電極/Dirichlet辺ごとに γ を指定(0=無効)。イオン衝突時に
  確率 γ で内向き法線方向に電子を放出(初期エネルギー指定可)。γモード放電の再現用
- サイクル: ①電荷堆積(P1形状関数=重心座標の重みで節点へ散布)→ ②ポアソン求解(行列は固定、prefactorize済みLUで右辺のみ更新)→ ③E補間 → ④リープフロッグでプッシュ → ⑤境界処理・粒子注入
- マクロ粒子: 重み付き超粒子。注入モデル(エミッタからの定常注入、初期一様プラズマ)
- 時間刻み: ωp·Δt < 0.2、要素横断制限(粒子が1ステップで隣接要素を大きく越えない)をチェックし警告
- 診断: 全エネルギー(場+運動)履歴、粒子数、位相空間プロット(x-vx等)、密度マップ
- 実行制御: WebSocketで進捗・中間フレームをフロントへストリーム、途中停止・再開
- 性能方針: まず numpy 一括処理で正しく → プロファイル → 電荷堆積とプッシュを CuPy 化

```jsonc
{
  "version": 1,
  "unit": "m",
  "geometry": {
    "domain": { "polygon": [[0,0],[0.1,0],[0.1,0.1],[0,0.1]] },

      { "id": "wire",  "type": "conductor", "shape": { "kind": "circle", "center": [0.05, 0.02], "radius": 0.005 }, "voltage": -200.0 },
      { "id": "diel1", "type": "dielectric", "polygon": [...], "eps_r": 4.0 },
      { "id": "beam",  "type": "charge",     "polygon": [...], "rho": -1e-6 }

    "boundaries": [
      { "edges": [0, 1], "type": "dirichlet", "voltage": 0.0 }
    ]
  },
```

## 13. 非機能要件

- フェーズ1の目標性能: 10万要素の求解+表示更新を数秒以内(CPU)
- すべての解析機能はGUIなしでも Python から呼べること(`es_sim` をライブラリとして import 可能に保つ。パラメータスイープ用)
- 自動テスト: ソルバーは解析解比較を必須とする。CI導入は任意
- クラッシュ耐性: 解析はバックエンドプロセス内で完結し、失敗してもUIは巻き込まれない

## 14. 既知のリスクと対応

| リスク | 対応 |
|---|---|
| FEM-PIC の実装難度(粒子位置特定・電荷堆積) | フェーズ2で walk探索・重心座標補間を先に固め、PICで再利用 |
| Windows での GPU 制約 | numpy で全機能完結。CuPy は NVIDIA 限定のオプションと割り切る |
| gmsh の配布サイズ・ライセンス(GPL) | 個人利用のため問題なし。配布する場合は Triangle 等への差し替えを検討 |
| JSON転送のオーバーヘッド | メッシュ10万要素程度までは実測上許容の見込み。超えたらバイナリ化 |
| Tauri(Rust)ビルド環境 | シェルは薄く保ち、Rustコードはほぼ触らない構成にする |
